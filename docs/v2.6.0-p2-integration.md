## 二、三波理论

### 核心定义

| 波次 | 涨幅（从低点） | 涨停次数 | 20日涨幅 | B1 策略 |
|------|---------------|---------|---------|---------|
| **建仓波** | 25% ~ 50% | ≤1 | 温和 | ✅ 可干，高概率 |
| **拉升波** | >50% 或快速脱离 | ≥2 | >30% | ⏸️ 等回调或 SB1 |
| **冲刺波** | >100% | ≥3 | >30% | ❌ 不看 |

### 算法逻辑

1. **找近期低点**：在 120 日 K 线中找局部最小值（连续 5 日最低点）
2. **统计区间数据**：
   - 累计涨幅、涨停次数、阳线占比
   - 日均涨幅、20 日涨幅
3. **评分制判定**：
   - 冲刺波优先判断（涨幅>100% + 频繁涨停）
   - 拉升波次之（涨幅>50% 或 20日>30% + 涨停≥2）
   - 建仓波兜底（25%≤涨幅≤50% + 涨停≤1 + 阳线>60%）

### 接口

```python
from modules.indicators import detect_three_waves, classify_wave_for_b1

result = detect_three_waves(klines)  # klines: List[DailyData]
# 返回：
# {
#   'wave': '建仓波' | '拉升波' | '冲刺波' | '未知',
#   'confidence': 0.85,
#   'stats': { 'gain_pct': 35.2, 'limit_up_count': 0, ... },
#   'b1_suggestion': '可干' | '等回调' | '不看' | '观望'
# }

suggestion = classify_wave_for_b1(klines)  # 简化接口，直接返回 B1 建议
```

## 三、麒麟会四阶段

### 核心定义

| 阶段 | 价格位置 | 成交量 | 形态特征 | 操作 |
|------|---------|--------|---------|------|
| **吸筹** | 相对 120 日低点 <30% | 放量（>60日均量 1.3x） | N型抬高、红肥绿瘦、低点抬升 | 关注，等 B1 |
| **拉升** | 已从低点涨 >30% | 量价齐升 | 快速脱离、有涨停、白线在上 | 不追，等回调 |
| **派发** | 相对 120 日高点 <15% | 高位放量、绿肥红瘦 | 出货五式/S1/S2/S3 信号 | 准备走人 |
| **回落** | 从高点跌 >20% | 缩量（<60日均量 0.8x） | 无承接、跌破均线 | 不抄底 |

### 子类型判断

| 类型 | 特征 | 典型盘口 |
|------|------|---------|
| **铁蝴蝶** | 拉升快、洗盘狠、出货 A 杀 | 对倒诱多、突然核按钮 |
| **学院派铁蝴蝶** | 拉升慢而稳、缩量回调、多重顶 | 大单托底、小单吃货 |

### 评分制算法

每个阶段独立打分（0-100），取最高分：

- **吸筹分** = 低位权重(30) + 放量权重(20) + N型权重(20) + 红肥绿瘦权重(20) + 无涨停权重(10)
- **拉升分** = 涨速权重(25) + 涨停权重(20) + 量价齐升权重(15) + 白线在上权重(10) + 呼吸节奏权重(10)
- **派发散** = 高位权重(30) + 出货信号权重(30) + 绿肥红瘦权重(20)
- **回落分** = 跌幅权重(30) + 缩量权重(25) + 无承接权重(20) + 跌破均线权重(15)

### 接口

```python
from modules.indicators import detect_kirin_stage

result = detect_kirin_stage(klines)  # klines: List[DailyData]
# 返回：
# {
#   'stage': '吸筹' | '拉升' | '派发' | '回落' | '未知',
#   'confidence': 0.72,
#   'sub_type': '铁蝴蝶' | '学院派铁蝴蝶' | '未知',
#   'scores': { 'xishou': 65, 'lasheng': 45, 'paifa': 20, 'luoluo': 10 },
#   'indicators': { 'price_position': '低位', 'vol_pattern': '放量', ... },
#   'operation': '关注，等B1' | '不追，等回调B1' | '准备走人' | '不抄底'
# }
```

## 四、策略集成

### 1. `modules/strategies.py`

在 `detect_all_strategies()` 的 **P1 指标全局检测** 之后，自动检测三波和麒麟会：

```python
# 三波理论信号
wave = detect_three_waves(daily_klines)
# 建仓波 → StrategyType.B1 + BUY
# 拉升波 → StrategyType.WATCH + HOLD
# 冲刺波 → StrategyType.S1 + SELL

# 麒麟会信号
kirin = detect_kirin_stage(daily_klines)
# 吸筹 → StrategyType.XISHOU
# 拉升 → StrategyType.LASHENG
# 派发 → StrategyType.PAIFA
# 回落 → StrategyType.LUOLUO
```

### 2. `modules/screener.py`

#### `analyze_stock()` 评分调整

三波/麒麟会结果直接影响综合评分：

| 条件 | 评分调整 |
|------|---------|
| 建仓波 | ×1.05 |
| 吸筹阶段 | ×1.08 |
| 冲刺波 | ×0.7 |
| 派发阶段 | ×0.7 |

同时风险分也会调整（冲刺波-20，派发-30，回落-15）。

#### `screen_stocks()` 新增选股条件

```python
# 建仓波选股
screen_stocks(criteria="build_wave")

# 吸筹阶段选股
screen_stocks(criteria="xishou")

# 安全选股（过滤冲刺波 + 派发/回落）
screen_stocks(criteria="safe")
```

## 五、使用示例

### Python API

```python
from modules.indicators import get_kline_data, detect_three_waves, detect_kirin_stage
from modules.indicators.core import DailyData

# 获取数据
klines = get_kline_data("600519.SH", days=120)
daily = [DailyData(**k) for k in klines]

# 三波理论
wave = detect_three_waves(daily)
print(f"当前处于: {wave['wave']}，B1建议: {wave['b1_suggestion']}")

# 麒麟会
kirin = detect_kirin_stage(daily)
print(f"主力阶段: {kirin['stage']}，操作建议: {kirin['operation']}")

# 综合判断
if wave['wave'] == '建仓波' and kirin['stage'] == '吸筹':
    print("✅ 高概率买点：建仓波 + 吸筹阶段")
elif wave['wave'] == '冲刺波' or kirin['stage'] == '派发':
    print("❌ 回避：处于高位或派发阶段")
```

### CLI 选股

```bash
# 筛选建仓波股票（B1可干）
python -m modules.cli screen --strategy 建仓波 --limit 10

# 筛选吸筹阶段股票
python -m modules.cli screen --strategy 吸筹 --limit 10

# 安全选股（排除冲刺波和派发）
python -m modules.cli screen --strategy 安全 --limit 20
```

## 六、注意事项

1. **数据量要求**：三波理论需要 ≥30 根 K 线，麒麟会需要 ≥60 根 K 线
2. **置信度阈值**：三波 ≥0.5，麒麟会 ≥0.4 才生成信号
3. **子类型判断**：仅在阶段明确为"拉升"或"派发"时进行
4. **评分调整**：选股时三波/麒麟会权重为辅助，不替代原有 B1/趋势/量价/风险评分
5. **边界处理**：涨幅在 45%-55% 之间可能同时满足建仓波和拉升波条件，以评分制取最高分为准
